# 图片处理工具功能扩展设计文档

## 一、概述

本文档为文件整理工具的图片处理模块功能扩展设计，在现有基础上增加实用类功能，提升用户工作效率。

### 现有功能回顾
- 图片查看：缩放、旋转、翻转
- 图片编辑：色彩调整、滤镜、裁剪
- 格式转换：JPEG/PNG/WebP/BMP/TIFF
- 批量压缩
- AI智能分类
- 相似照片检测

---

## 二、新增功能设计

### 2.1 批量重命名

#### 功能定位
帮助用户批量修改文件名，支持多种命名规则，适用于整理照片、截图等场景。

#### 核心特性
1. **命名规则**
   - 序号递增：支持起始序号、序号位数（如 001, 002）
   - 日期前缀：支持从文件EXIF日期、修改日期、创建日期提取
   - 文字替换：查找替换文件名中的特定文字
   - 字符处理：移除特殊字符、空格转下划线、大小写转换
   - 前缀/后缀：添加固定前缀或后缀

2. **预览机制**
   - 实时预览重命名效果
   - 高亮显示变更部分
   - 冲突检测（重复文件名）

3. **执行选项**
   - 覆盖原文件
   - 复制到新位置重命名
   - 移动到新位置重命名

#### 交互流程
```
选择文件 → 设置规则 → 预览结果 → 确认执行
```

#### UI 设计
- 左侧文件列表（支持多选）
- 中间规则配置面板（Tab切换不同规则类型）
- 右侧实时预览（Before/After 对比）

---

### 2.2 图片水印

#### 功能定位
为图片添加文字或图片水印，支持批量处理，适用于版权保护。

#### 核心特性
1. **水印类型**
   - 文字水印：自定义文字、字体、大小、颜色、透明度
   - 图片水印：支持PNG图片（带透明度）

2. **位置控制**
   - 九宫格位置选择（左上/中上/右上...）
   - 自定义坐标偏移
   - 平铺水印（用于版权保护）

3. **批量处理**
   - 多图批量添加水印
   - 统一配置应用到所有图片
   - 支持按原图尺寸自适应水印大小

#### 参数配置
| 参数 | 说明 | 范围 |
|------|------|------|
| 文字内容 | 水印文字 | 用户输入 |
| 字体大小 | 文字大小 | 12-200px |
| 字体颜色 | 文字颜色 | 颜色选择器 |
| 透明度 | 水印透明度 | 0-100% |
| 位置 | 水印位置 | 九宫格 |
| 边距 | 距边缘距离 | 0-100px |

---

### 2.3 图片拼接

#### 功能定位
将多张图片拼接为一张，支持横向、纵向拼接，适用于截图拼接、漫画合成。

#### 核心特性
1. **拼接模式**
   - 横向拼接：左到右并排
   - 纵向拼接：上到下依次
   - 网格拼接：指定行列数自动排布

2. **拼接设置**
   - 间距设置：图片之间的间距
   - 边框设置：是否添加边框及颜色
   - 背景颜色：拼接触发背景色
   - 对齐方式：上/中/下 或 左/中/右

3. **图片排序**
   - 拖拽调整顺序
   - 按文件名排序
   - 按日期排序

#### 输出设置
- 输出尺寸：自动计算或自定义
- 输出格式：JPEG/PNG/WebP
- 输出质量：1-100

---

### 2.4 GIF 制作

#### 功能定位
将多张图片合成为 GIF 动画，适用于制作表情包、简单动画。

#### 核心特性
1. **帧设置**
   - 帧顺序：拖拽排序
   - 帧延迟：每帧显示时长（毫秒）
   - 循环次数：一次/无限循环

2. **预览**
   - 实时预览动画效果
   - 逐帧预览

3. **优化**
   - 尺寸调整
   - 颜色优化
   - 文件大小预估

---

### 2.5 图片类型转换

#### 功能定位
将单张或多张图片转换为多种格式，包括常见图像类型和 PDF。在选择 PDF 时，所有图片会按列表顺序合并为一个文档。

#### 核心特性
1. **转换模式**
   - 图像格式转换：JPEG、PNG、WebP、BMP、TIFF 等
   - 合并为 PDF：当目标格式为 PDF 时，图片列表顺序决定页序

2. **页面设置**
   - 页面尺寸：A4/A3/Letter/原图尺寸/自定义
   - 页面方向：自动/纵向/横向
   - 边距设置：页边距大小

3. **图片排列**
   - 每页一张：原图比例放入
   - 每页多张：2x2, 2x3 等网格排列

---

### 2.6 缩略图生成

#### 功能定位
为指定目录生成统一规格的缩略图，适用于整理照片库、制作图册。

#### 核心特性
1. **尺寸设置**
   - 预设尺寸：320x240, 640x480, 800x600, 1024x768, 1920x1080
   - 自定义尺寸
   - 保持比例/裁剪/填充

2. **输出设置**
   - 输出目录：原目录/指定目录
   - 文件命名：原名_thumb, thumb_原名, 自定义
   - 覆盖策略：跳过/覆盖/重命名

3. **质量设置**
   - 输出格式：JPEG/PNG/WebP
   - 压缩质量：1-100

---

### 2.7 图片增强

#### 功能定位
一键优化图片质量，适用于提升旧照片、模糊照片的清晰度。

#### 核心特性
1. **自动增强**
   - 智能调整：自动平衡曝光、对比度、色彩
   - 超分辨率：2x/4x 放大同时提升清晰度
   - 去雾：去除照片雾霾

2. **手动微调**
   - 清晰度增强
   - 降噪处理
   - 锐化处理

```
                           ↓

│                    主进程处理层                         │
├─────────────────────────────────────────────────────────┤
│  参数校验 → 执行处理 → 结果返回 → 文件系统操作           │
└─────────────────────────────────────────────────────────┘
```

### 5.2 批量处理流程

```typescript
// 批量处理通用模式
async function executeBatchOperation<T, R>(
  files: string[],
  operation: (file: string) => Promise<R>,
  options: {
    onProgress?: (current: number, total: number) => void
    onFileComplete?: (result: R) => void
    concurrency?: number  // 并发数，默认3
    stopOnError?: boolean
  }
): Promise<R[]> {
  const results: R[] = []
  const total = files.length
  
  for (let i = 0; i < total; i++) {
    try {
      const result = await operation(files[i])
      results.push(result)
      options.onProgress?.(i + 1, total)
      options.onFileComplete?.(result)
    } catch (error) {
      if (options.stopOnError) {
        throw error
      }
      // 记录错误但继续执行
    }
  }
  
  return results
}
```

### 5.3 进度追踪

所有批量操作均支持进度追踪：

## 八、优先级与版本规划

### Phase 1: 基础功能（推荐优先实现）
1. **批量重命名** - 需求最高，实现相对简单
2. **图片拼接** - 常用功能，技术成熟
3. **图片转PDF** - 实用性强，需求广泛

### Phase 2: 增强功能
4. **添加水印** - 版权保护需求
5. **缩略图生成** - 图片整理需求
6. **GIF制作** - 趣味性功能

### Phase 3: 高级功能
7. **图片增强** - 需要引入AI模型（如Real-ESRGAN）

### 9.1 类型定义补充

```typescript
// types/imageTools.ts

export interface BatchRenameOptions {
  mode: 'sequence' | 'date' | 'replace' | 'prefix' | 'suffix'

  dateFormat?: string
  findText?: string
  replaceText?: string
  prefix?: string
  suffix?: string
  caseSensitive?: boolean
  outputPath?: string

  type: 'text' | 'image'
  text?: {
    content: string
    fontSize: number
    fontFamily: string
    color: string
    opacity: number
  }
  image?: {
    path: string
    scale: number
    opacity: number
  }
  position: 'top-left' | 'top-center' | 'top-right' | ... | 'bottom-right'

export interface StitchOptions {
  mode: 'horizontal' | 'vertical' | 'grid'
  rows?: number
  cols?: number
  gap: number
  backgroundColor: string
  align: 'start' | 'center' | 'end'
  outputFormat: 'jpeg' | 'png' | 'webp'
  quality: number
}

export interface GifOptions {
  width?: number
  height?: number
  delay: number
  loop: number
  quality: number
}

export interface PdfOptions {
  pageSize: 'a4' | 'a3' | 'letter' | 'original'
  orientation: 'auto' | 'portrait' | 'landscape'
  margin: number
  imagesPerPage: number | 'auto'
  outputPath: string
}

export interface ThumbnailOptions {
  width: number
  height: number
  fit: 'cover' | 'contain' | 'fill'
  format: 'jpeg' | 'png' | 'webp'
  quality: number
  outputDir: 'same' | 'custom'
  naming: 'prefix' | 'suffix' | 'custom'

  suffix?: string

export interface EnhanceOptions {

    saturation: number
    sharpness: number

  }
  scale?: 1 | 2 | 4
```

### 9.2 依赖库推荐
- **GIF制作**: `gifenc` 或 `gif.js`（纯JS，无需额外native依赖）
- **PDF生成**: `pdfkit` 或 `sharp` 内置PDF支持
- **图片增强**: 考虑集成 `sharp` 高级功能或 WebAI 模型
